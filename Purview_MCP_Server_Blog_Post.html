What is MCP and Why Should You Care?

Imagine telling an AI assistant: "Search all Copilot interactions from the last 6 months, generate an analytics report, and upload it to SharePoint" — and it just does it. No scripts. No portals. No clicking through 47 tabs.

That's what Model Context Protocol (MCP) makes possible. MCP is an open standard created by Anthropic that lets AI assistants like Claude and Microsoft Copilot Studio connect to external tools and APIs through a standardized interface.

🎯 What This Server Does

The Purview MCP Server gives your AI assistant 106 tools to manage:

🔍 Audit Search

Search any M365 workload audit log — Exchange, SharePoint, Teams, Entra ID, Copilot, DLP

📁 SharePoint

Sites, lists, files, sharing, permissions, pages — full CRUD

🤖 Copilot Tracking

Agent usage, security scans, trend analysis, HTML dashboards

📊 Threat Hunting

10 MITRE ATT&CK presets, risk scoring, executive summaries

🏷️ Sensitivity Labels

Apply, remove, scan labels at site/library/file level

🛡️ Security

Alerts, incidents, Secure Score, eDiscovery, identity risk, KQL

📋 35 Workload Presets

One-click audit search for any M365 service

🏥 Health & Reports

Service health, usage reports, sign-in logs, Conditional Access

Create Azure App Registration

  1. Go to portal.azure.com and sign in as an admin
  2. Search for App registrations in the top search bar
  3. Click + New registration
  4. Name: Purview MCP Server
  5. Supported account types: Single tenant
  6. Redirect URI: leave blank → Click Register
  7. On the Overview page, copy the Application (client) ID
  8. Also copy the Directory (tenant) ID
⚠️
Save both IDs! You'll need them for the MCP server configuration. Keep them somewhere safe.

🔑 Create Client Secret

  1. In your app registration, click Certificates & secrets
  2. Click + New client secret
  3. Description: Purview MCP → Expires: 24 months
  4. Click Add
  5. IMMEDIATELY copy the Value — you can only see it once!
🔴
Copy the secret NOW. Once you leave this page, it's gone forever and you'll need to create a new one.

Install & Build the MCP Server

📋 Prerequisites

  1. Node.js 20+ — download from nodejs.org. Verify: node --version
  2. Azure CLI — download from aka.ms/installazurecliwindows (only for Azure deploy)

🚀 Build Steps

# 1. Extract the purview-mcp-server.zip to C:\Projects\
cd C:\Projects\purview-mcp-server

# 2. Install dependencies
npm install

# 3. Build TypeScript
npm run build

# 4. Verify — should see 12 .js files in dist/
dir dist\*.js
✅
If npm run build completes with no errors, you're ready! The compiled JavaScript lives in the dist/ folder.

🧪 Quick Test (Optional)

# Set environment variables temporarily
set PURVIEW_TENANT_ID=YOUR_TENANT_ID
set PURVIEW_CLIENT_ID=YOUR_CLIENT_ID
set PURVIEW_CLIENT_SECRET=YOUR_SECRET

# Run the server
node dist/index.js

# Should print: [Purview MCP] Server started successfully
# Press Ctrl+C to stop

Configure Claude Desktop

  1. Open the file %APPDATA%\Claude\claude_desktop_config.json
  2. Add the Purview MCP server configuration (see below)
  3. Create the export folder: C:\Projects\PurviewExports
  4. Fully quit Claude Desktop (right-click system tray → Quit) and reopen it
{
  "mcpServers": {
    "purview": {
      "command": "node",
      "args": ["C:\\Projects\\purview-mcp-server\\dist\\index.js"],
      "env": {
        "PURVIEW_TENANT_ID": "ENTER_YOUR_TENANT_ID_HERE",
        "PURVIEW_CLIENT_ID": "ENTER_YOUR_CLIENT_ID_HERE",
        "PURVIEW_CLIENT_SECRET": "ENTER_YOUR_SECRET_HERE",
        "PURVIEW_EXPORT_PATH": "C:\\Projects\\PurviewExports"
      }
    }
  }
}
💡
Replace ENTER_YOUR_* with the values from Step 3. If you already have other MCP servers configured, add "purview" inside the existing mcpServers block.

Deploy to Azure App Service

Deploying to Azure lets you connect the MCP server to Copilot Studio so your agents can use all 106 tools in the cloud.

🚀 One-Command Deploy

cd C:\Projects\purview-mcp-server

.\deploy-azure.ps1 `
  -TenantId "YOUR_TENANT_ID" `
  -ClientId "YOUR_CLIENT_ID" `
  -ClientSecret "YOUR_SECRET"

The script does everything: login, create resources, build, package with node_modules, deploy, set startup command, verify health.

🔧 Key Azure Fixes Built Into the Script

Connect to Copilot Studio

After deploying to Azure, connect your Copilot Studio agent to the MCP server:

  1. Go to copilotstudio.microsoft.com
  2. Open your agent (or create a new one)
  3. Click Tools → Add a tool
  4. Click New tool → select Model Context Protocol
  5. Paste your MCP URL: https://YOUR-APP.azurewebsites.net/mcp
  6. Auth: None → Click Create
  7. Select all 106 tools → Click Add
  8. Go to Settings → Generative AI → Turn ON Generative Orchestration
✅
Your Copilot agent now has access to all 106 Purview tools! Ask it: "Search for Copilot interactions from last 7 days" and it will call the MCP tools automatically.
⚠️
Generative Orchestration must be ON or MCP tools won't be called dynamically. The agent needs this to decide when to use which tool.
